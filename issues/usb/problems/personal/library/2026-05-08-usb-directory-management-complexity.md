# USBディレクトリ管理が複雑化する

## 概要

- 対象ツール: USB / Mixxx / Serato
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

USB内の音源をディレクトリだけで整理しようとすると、アルバム別、用途別、リスト別、DJソフト別の観点が混ざりやすい

その結果、重複配置、タグ管理、フォーマット混在、低使用頻度曲の滞留、バックアップ対象の判断が同じ問題として絡み合う状態

## 再現条件

- USB内に多数の音源ファイルを配置
- アルバム別、イベント別、用途別などでディレクトリを分ける運用
- Mixxx と Serato の両方で同じUSB内の音源を利用
- MP3を中心にしつつ、一部 WAV などの別フォーマットが混在
- MP3タグが未整備、またはファイル名とタグ情報が一致していない状態
- 使用頻度の低い曲をUSB内に残し続ける運用

## 期待する状態

- USB内のディレクトリ構造とDJソフト側プレイリストの役割が分かれている状態
- 個別の問題を1つずつ確認、対応できる状態
- USB運用の基準が、重複、タグ、フォーマット、利用頻度、バックアップの観点で整理されている状態

## 現時点の回避策

- MP3を基本フォーマットとして扱う
- MP3Tag などで曲名、アーティスト名、アルバム名などのタグを整える
- アルバム別、リスト別のディレクトリコピーを増やしすぎない
- 低使用頻度曲はUSB外のアーカイブへ退避する
- USB内のディレクトリ構造とDJソフト側プレイリストのどちらを正とするかを決める

## 個人製作アプリで対応できそうな範囲

- 分割した各問題の検出、一覧化、候補生成
- USB内ファイルの棚卸し結果にもとづく運用ルールの確認

## 対応が難しそうな範囲

- Mixxx と Serato の内部仕様差そのものの解消
- DJソフト固有の解析結果、キュー、ループ情報の完全互換
- タグ情報が存在しない音源の正確な自動補完
- 本番利用時のUSB物理故障の完全防止

## 調査メモ

- USBディレクトリだけで分類を完結させると、プレイリスト管理と二重化しやすい
- ファイル移動により、Mixxx や Serato 側の参照パスが切れる可能性
- WAV は容量が大きく、タグ管理もMP3より不安定になりやすい
- 日本語、特殊記号、文字コード、パス長の問題も確認が必要
- BPM、キー、波形、キュー、ループなどの解析済み情報をどこまでUSB運用に含めるか検討が必要
