# DJソフトごとに扱いやすいディレクトリ構造が異なる

## 概要

- 対象ツール: USB / Mixxx / Serato
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

Mixxx と Serato ではUSB内のディレクトリやライブラリ管理に対する前提が異なるため、片方で扱いやすい構成がもう片方では扱いにくくなる可能性

同じUSBを複数ツールで使おうとすると、フォルダ構成、プレイリスト、解析済み情報のどれを基準にするかが曖昧になりやすい

## 再現条件

- Mixxx と Serato の両方で同じUSB内の音源を利用
- USB内のフォルダ構成をDJソフト側の整理単位として扱う
- 片方のDJソフトで作成したライブラリやリストをもう片方でも利用したい場合

## 期待する状態

- Mixxx と Serato のどちらでも破綻しにくいディレクトリ構造
- USB内の物理配置とDJソフト側プレイリストの責務が分かれている状態
- ツール固有情報に依存しない共通の音源配置ルールがある状態

## 現時点の回避策

- USB内はできるだけ単純なフォルダ構成にする
- DJソフト固有のリスト管理を物理ディレクトリと混ぜすぎない
- 片方のDJソフトだけに最適化した構成を避ける

## 個人製作アプリで対応できそうな範囲

- Mixxx と Serato の両方で扱いやすいディレクトリ構造案の生成
- USB内ファイルの配置ルールチェック
- DJソフト別の参照パス差分確認
- 共通プレイリスト用の中間リスト生成

## 対応が難しそうな範囲

- Mixxx と Serato の内部仕様差そのものの解消
- DJソフト固有の解析結果、キュー、ループ情報の完全互換
- ツール側の読み込み仕様変更への自動追従

## 調査メモ

- ファイル移動により、Mixxx や Serato 側の参照パスが切れる可能性
- ディレクトリ構造を正にするか、DJソフト側プレイリストを正にするかの判断が必要
- 解析済み情報をどこまでUSB運用に含めるか検討が必要
