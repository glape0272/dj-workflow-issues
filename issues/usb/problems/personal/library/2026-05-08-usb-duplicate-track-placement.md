# USB内で同じ曲が複数箇所に配置される

## 概要

- 対象ツール: USB / Mixxx / Serato
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

USB内の音源ディレクトリをアルバム別、イベント別、用途別、リスト別に分けると、同じ曲を複数箇所へコピーしやすい

重複が増えると容量を消費するだけでなく、片方だけタグ修正や差し替えが行われ、同じ曲のはずなのに状態が揃わない可能性

## 再現条件

- USB内に多数の音源ファイルを配置
- アルバム別、用途別、イベント別など複数の分類軸でフォルダを作成
- プレイリスト代わりにディレクトリコピーを利用
- 同じ曲を複数のセットや用途で使う運用

## 期待する状態

- USB内の音源ファイルを重複させずに管理できる状態
- 1つの実体ファイルを正として扱える状態
- プレイリストや分類情報とファイル配置の役割が分かれている状態

## 現時点の回避策

- アルバム別、リスト別のディレクトリコピーを増やしすぎない
- USB内のディレクトリ構造とDJソフト側プレイリストのどちらを正とするかを決める
- 低使用頻度のセット用コピーを定期的に棚卸しする

## 個人製作アプリで対応できそうな範囲

- USB内の重複曲検出
- ファイルサイズ、ハッシュ、タグ情報にもとづく同一候補の一覧化
- 重複配置されているディレクトリの可視化
- 残すファイルと削除候補の一覧生成

## 対応が難しそうな範囲

- DJソフト側プレイリストの意図を完全に推測すること
- 音源の別バージョン、編集版、リマスター版の自動判定
- 本番利用前に削除してよいファイルかどうかの最終判断

## 調査メモ

- ファイル名だけでは同一曲判定が不安定
- タグ情報と音声ハッシュの併用が必要になる可能性
- ディレクトリ分類を増やすほどプレイリスト管理と二重化しやすい
