# MixxxとSeratoとUSB間でライブラリ状態を共有しにくい

## 概要

- 対象ツール: Mixxx / Serato / USB
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

Mixxx と Serato はそれぞれライブラリ状態の持ち方が異なり、USBや別PCをまたいで解析結果、キュー、プレイリスト、参照パスを共有しにくい

音源ファイルだけを外部ストレージに置いても、DJソフト側の管理情報が環境ごとに分断される状態

## 再現条件

- Mixxx と Serato の両方で同じ音源を使う
- 同じUSBや外部ストレージを複数PCで利用
- 解析済み情報、Cue、クレート、プレイリストを持ち回りたい
- PCごとにパスやライブラリ保存先が異なる

## 期待する状態

- 音源ファイルとライブラリ状態の依存関係が分かっている状態
- 共有できる情報と共有できない情報が切り分けられている状態
- 別PCやUSB運用時の再設定負荷が低い状態

## 現時点の回避策

- PCごとにライブラリ登録と解析を行う
- DJソフト固有の管理情報は手動で移行する
- USB内の音源配置をできるだけ安定させる

## 個人製作アプリで対応できそうな範囲

- 各ツールのライブラリ参照パス一覧化
- USB内ファイルとDJソフト側参照情報の差分確認
- 共有できる情報とできない情報の棚卸し
- 移行前チェックリスト生成

## 対応が難しそうな範囲

- Mixxx と Serato の内部ライブラリ仕様差の解消
- キュー、ループ、解析結果の完全互換
- 非公開仕様への安全な書き込み

## 調査メモ

- MixxxはDB、Seratoはクレートやライブラリ保存形式の確認が必要
- 音源配置、解析情報、分類情報を分けて整理する必要
- 共有可能性は読み取り、変換、書き込みで分けて評価したい
