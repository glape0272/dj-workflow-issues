# Spotify APIでは操作できるがDJソフト側の外部操作範囲が限られる

## 概要

- 対象ツール: Spotify / Serato / Mixxx
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

SpotifyはAPIでプレイリスト周りを操作できる可能性がある一方、SeratoやMixxxのライブラリ操作は外部アプリから安全に扱える範囲が異なる

外部アプリで整理を自動化したくても、ツールごとに読み取り、候補生成、書き込みの可否が分かれ、どこまで自動化できるか判断しにくい

## 再現条件

- Spotifyのプレイリスト整理を外部アプリで補助したい
- SeratoやMixxxにも分類結果を反映したい
- DJソフト側のライブラリやクレートを外部から扱いたい
- 安全に書き込める仕様が不明な状態

## 期待する状態

- ツールごとに読み取り可能、書き込み可能、操作不可の範囲が分かっている状態
- 自動化できる範囲と手動確認が必要な範囲が分かれている状態
- 変更前に差分を確認できる状態

## 現時点の回避策

- Spotify側はAPI利用を検討する
- SeratoやMixxx側はまず読み取りと差分確認に範囲を絞る
- 書き込みを伴う操作は手動反映を前提にする

## 個人製作アプリで対応できそうな範囲

- ツール別API / ファイル形式 / DBの調査結果整理
- 読み取り専用の棚卸しと候補生成
- Spotifyプレイリスト操作の補助
- Serato / Mixxx向けの手動反映リスト生成

## 対応が難しそうな範囲

- 非公開仕様への安全な直接書き込み
- すべてのツールで同じ操作を自動化すること
- DJソフト側の仕様変更に完全追従すること

## 調査メモ

- 操作は「取得」「候補生成」「反映」に分けて設計する必要
- 反映は最初から完全自動にせず、確認後反映が安全
- Spotify APIでできる範囲とDJソフト側でできる範囲の差を明文化したい
