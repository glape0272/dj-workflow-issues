# 曲数増加で全ツール横断の棚卸し対象が膨らむ

## 概要

- 対象ツール: Spotify / USB / Serato / Mixxx
- 発生日: 2026-05-08
- 優先度: 中
- 状態: 未調査
- 出どころ: personal

## 困っていること

Spotifyのお気に入り、USB内音源、Seratoライブラリ、Mixxxライブラリがそれぞれ増えると、棚卸し対象がツールごとに膨らむ

曲数増加により、重複確認、未分類確認、タグ修正、参照切れ確認、低使用頻度曲の退避が全体的に重くなる

## 再現条件

- Spotifyのお気に入りが大量にある
- USB内の音源数が増える
- SeratoやMixxxのライブラリ曲数が増える
- 複数ツールで同じ曲を扱う

## 期待する状態

- ツール横断で棚卸し対象を絞り込める状態
- 未分類、重複、低使用頻度、参照切れをまとめて確認できる状態
- 常用曲とアーカイブ曲を分けて管理できる状態

## 現時点の回避策

- ツールごとに手動で整理する
- 低使用頻度曲をUSB外へ退避する
- お気に入りやライブラリを追加日順に確認する

## 個人製作アプリで対応できそうな範囲

- ツール横断の棚卸し対象リスト生成
- 未分類曲、重複候補、低使用頻度候補の抽出
- Spotify / USB / DJソフトの存在状況一覧化
- 整理優先度の候補生成

## 対応が難しそうな範囲

- 今後使う曲かどうかの完全自動判断
- DJ用途における価値判断の完全代替
- 各ツール内部状態の完全自動整理

## 調査メモ

- 棚卸し単位を曲単位、ファイル単位、リスト所属単位に分ける必要
- 低使用頻度曲は削除ではなく退避候補として扱うほうが安全
- 横断ダッシュボードの中心課題になりそう
